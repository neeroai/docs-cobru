## Prerrequisitos

- Cuenta Cobru con credenciales desde [panel.cobru.co](https://panel.cobru.co)
- Tres valores de credenciales: `Api-Token`, `Api-Secret-Key`, `refresh_token`

## Autenticación

Cobru usa un sistema de auth de dos capas: API keys estáticas + token de acceso de corta duración.

### Paso 1 — Obtener access token

```http
POST {COBRU_BASE_URL}/token/refresh/
Api-Token: {tu_api_token}
Api-Secret-Key: {tu_api_secret}
Content-Type: application/json

{ "refresh": "{tu_refresh_token}" }
```

<Callout type="warn">
  El campo es `refresh`, **no** `refresh_token`. La documentación del plugin WooCommerce es incorrecta en este punto.
</Callout>

**Respuesta `200`:**
```json
{ "access": "eyJ0eXAi..." }
```

El token `access` tiene un TTL de ~60 minutos. Guárdalo en caché máximo 50 minutos.

### Paso 2 — Incluir las tres credenciales en requests siguientes

```http
POST {COBRU_BASE_URL}/cobru/
Api-Token: {tu_api_token}
Api-Secret-Key: {tu_api_secret}
Authorization: Bearer {access}
Content-Type: application/json
```

## Crear un pago

```json
{
  "amount": 50000,
  "description": "Pago de servicio",
  "expiration_days": 1,
  "client_assume_costs": false,
  "iva": 0,
  "payment_method_enabled": "{\"Bre-B\":true,\"bancolombia_qr\":true}",
  "payer_redirect_url": "https://tuapp.com/pago-exitoso",
  "callback": "https://tuapp.com/api/cobru/webhook"
}
```

<Callout type="warn">
  `payment_method_enabled` debe ser un **string JSON serializado** — ejecuta `JSON.stringify({...})` antes de incluirlo.
  Enviarlo como objeto plano devuelve HTTP 400.
</Callout>

**Respuesta `201`:**
```json
{
  "pk": 27150,
  "url": "3gofdf6f",
  "amount": "50000.00",
  "state": 0,
  "fee_amount": 2395,
  "currency_code": "COP"
}
```

### Construir la URL de pago

```typescript
// CORRECTO
const paymentUrl = `${COBRU_BASE_URL}/${response.url}`;
// Ejemplo: https://dev.cobru.co/3gofdf6f

// INCORRECTO (patrón del plugin WooCommerce — NO funciona)
const paymentUrl = `${COBRU_BASE_URL}/pay/${response.pk}`;
```

## Manejar webhooks

Cobru hace POST a tu URL `callback` cuando cambia el estado del pago:

```json
{
  "orderId": "...",
  "state": 3,
  "payment_method": "Bre-B",
  "amount": "50000.00",
  "url": "3gofdf6f"
}
```

| Estado | Significado |
|--------|-------------|
| `0` | Pendiente |
| `1` | Procesando |
| `2` | Rechazado |
| `3` | Aprobado |
| `4` | Reembolsado |

<Callout type="info">
  Cobru no firma los callbacks webhook (sin HMAC ni JWT). Siempre responde `200` para evitar reintentos.
  Para seguridad, valida via IP allowlist o un secreto en el query param del callback.
</Callout>

## Variables de entorno

```bash
COBRU_BASE_URL=https://dev.cobru.co   # sandbox
COBRU_API_TOKEN=tu_api_token
COBRU_API_SECRET=tu_api_secret_key
COBRU_REFRESH_TOKEN=tu_refresh_token
NEXT_PUBLIC_APP_URL=https://tuapp.com
```
