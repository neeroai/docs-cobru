## ¿Qué es BRE-B?

Sistema de Pagos Inmediatos operado por Redeban (Banco de la República de Colombia). Permite transferencias bancarias instantáneas entre cuentas colombianas mediante QR interoperable. Lanzado en marzo 2024.

## Flujo de pago

```
Usuario → App → Cobru API → Generador QR → Usuario recibe QR
                  ↓
           Webhook callback en el pago
```

1. La app llama a la API de Cobru para crear un pago con `"Bre-B": true` en `payment_method_enabled`
2. Cobru devuelve `pk` (entero) y `url` (slug corto)
3. La app construye la URL de pago: `{COBRU_BASE_URL}/{url}`
4. La app genera un PNG QR desde la URL de pago
5. La app envía el QR al usuario (ej: via mensaje de imagen en WhatsApp)
6. El usuario escanea el QR → Cobru procesa el pago
7. Cobru hace POST al `callback` con `state: 3` al aprobar

## Generar PNG QR

Cobru no expone un endpoint de imagen QR. Usa un generador externo:

```typescript
const paymentUrl = `${COBRU_BASE_URL}/${cobruResponse.url}`;
const qrUrl = `https://api.qrserver.com/v1/create-qr-code/?size=512x512&data=${encodeURIComponent(paymentUrl)}&margin=10`;
```

`api.qrserver.com` devuelve `200 image/png` en menos de 100ms, sin autenticación.

<Callout type="info">
  El QR apunta a la página de pago de Cobru — el usuario ve el QR nativo de Bre-B más todos los otros métodos habilitados.
  Para QR EMV nativo directamente en tu app, contacta soporte@cobru.co — no está expuesto en la API actual.
</Callout>

## Habilitar BRE-B en un pago

```typescript
const body = {
  amount: 50000,
  description: 'Pago Bre-B',
  expiration_days: 1,
  client_assume_costs: false,
  iva: 0,
  // Debe ser JSON.stringify'd
  payment_method_enabled: JSON.stringify({
    'Bre-B': true,
    bancolombia_qr: true,
  }),
  payer_redirect_url: 'https://tuapp.com/pago-exitoso',
  callback: `https://tuapp.com/api/cobru/webhook?ref=${referenceId}`,
};
```

## Enviar QR por WhatsApp

```typescript
// Mensaje de imagen WhatsApp Cloud API
const payload = {
  messaging_product: 'whatsapp',
  to: '573001234567',
  type: 'image',
  image: {
    link: qrUrl, // URL de api.qrserver.com — WhatsApp lo descarga directamente
    caption: `Pago Bre-B/QR - $${amount.toLocaleString('es-CO')}\nAbrir: ${paymentUrl}`,
  },
};
```

WhatsApp Cloud API descarga el PNG directamente — no se necesita upload previo.

## Caché del token

| Propiedad | Valor |
|-----------|-------|
| TTL real del token de Cobru | ~60 min (inferido) |
| TTL recomendado para caché | 50 min |
| Edge Runtime | El caché en memoria no persiste entre invocaciones — 1 refresh por invocación |
| Fix para producción | Guardar token en Vercel KV / Redis con TTL de 50 min |

## Limitaciones conocidas

| Limitación | Impacto | Workaround |
|------------|---------|------------|
| Sin QR EMV nativo | El usuario ve la página web de Cobru, no QR bancario directo | Aceptable para MVP |
| Sin firma en webhook | Cualquiera podría hacer POST al callback | Validar via IP allowlist o secreto en query param |
| Sin endpoint de consulta de estado | No se puede hacer polling | Usar webhook; usar `idempotency_key` de la respuesta para deduplicación |
| `amount` en respuesta es string | Requiere parsing | `parseFloat(response.amount)` |
