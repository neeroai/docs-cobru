<Callout type="warn">
  El endpoint de balance viene de materiales legacy de Cobru. La ruta es útil, pero el contrato completo todavía necesita una validación fresca en sandbox.
</Callout>

## Endpoint

`GET /balance/`

## Usos típicos

- validaciones previas antes de envíos o retiros
- sincronización de saldo en dashboard
- vistas de tesorería y conciliación

## Respuesta observada en documentación legacy

```json
{
  "balance": "14179.25",
  "balance_cop": "14179.25",
  "balance_usd": "9995.00",
  "balance_ars": "10000.00",
  "balance_mxn": "96536.31",
  "balance_brl": "89772737.20"
}
```

## Guía de integración

<Accordions>
  <Accordion title="No trates la respuesta como verdad contable absoluta">
    Usa la lectura de balance como snapshot operativo, no como única fuente de verdad para settlement. Combínala con movimientos y estados de pagos o retiros.
  </Accordion>

  <Accordion title="Cachea con cuidado">
    Para dashboards es razonable cachear por poco tiempo. Para payouts o retiros, vuelve a consultar el saldo inmediatamente antes de ejecutar la operación.
  </Accordion>

  <Accordion title="Campos multi-moneda">
    La respuesta legacy incluye saldos en COP, USD, ARS, MXN y BRL. Confirma cuáles son realmente significativos para tu tipo de cuenta antes de mostrarlos en UI de producción.
  </Accordion>
</Accordions>
