## Prerequisites

- Cobru account with API credentials from [panel.cobru.co](https://panel.cobru.co)
- Three credential values: `Api-Token`, `Api-Secret-Key`, `refresh_token`

## Authentication

Cobru uses a two-layer auth system: static API keys + a short-lived access token.

### Step 1 — Get an access token

```http
POST {COBRU_BASE_URL}/token/refresh/
Api-Token: {your_api_token}
Api-Secret-Key: {your_api_secret}
Content-Type: application/json

{ "refresh": "{your_refresh_token}" }
```

<Callout type="warn">
  The field is `refresh`, **not** `refresh_token`. The WooCommerce plugin documentation is wrong on this point.
</Callout>

**Response `200`:**
```json
{ "access": "eyJ0eXAi..." }
```

The `access` token has a TTL of ~60 minutes. Cache it for at most 50 minutes to avoid expiry mid-request.

### Step 2 — Include all three credentials on subsequent requests

```http
POST {COBRU_BASE_URL}/cobru/
Api-Token: {your_api_token}
Api-Secret-Key: {your_api_secret}
Authorization: Bearer {access}
Content-Type: application/json
```

## Create a payment

```json
{
  "amount": 50000,
  "description": "Service payment",
  "expiration_days": 1,
  "client_assume_costs": false,
  "iva": 0,
  "payment_method_enabled": "{\"Bre-B\":true,\"bancolombia_qr\":true}",
  "payer_redirect_url": "https://yourapp.com/payment-success",
  "callback": "https://yourapp.com/api/cobru/webhook"
}
```

<Callout type="warn">
  `payment_method_enabled` must be a **JSON-serialized string** — run `JSON.stringify({...})` before including it in the body.
  Sending it as a plain object returns HTTP 400.
</Callout>

**Response `201`:**
```json
{
  "pk": 27150,
  "url": "3gofdf6f",
  "amount": "50000.00",
  "state": 0,
  "fee_amount": 2395,
  "currency_code": "COP"
}
```

### Construct the payment URL

```typescript
// CORRECT
const paymentUrl = `${COBRU_BASE_URL}/${response.url}`;
// Example: https://dev.cobru.co/3gofdf6f

// WRONG (WooCommerce plugin pattern — does NOT work)
const paymentUrl = `${COBRU_BASE_URL}/pay/${response.pk}`;
```

## Handle webhooks

Cobru POSTs to your `callback` URL when payment state changes:

```json
{
  "orderId": "...",
  "state": 3,
  "payment_method": "Bre-B",
  "amount": "50000.00",
  "url": "3gofdf6f"
}
```

| State | Meaning |
|-------|---------|
| `0` | Pending |
| `1` | Processing |
| `2` | Rejected |
| `3` | Approved |
| `4` | Refunded |

<Callout type="info">
  Cobru does not sign webhook callbacks (no HMAC or JWT). Always respond `200` to prevent retries.
  For security, validate via IP allowlist or a secret in the callback query param.
</Callout>

## Environment variables

```bash
COBRU_BASE_URL=https://dev.cobru.co   # sandbox
COBRU_API_TOKEN=your_api_token
COBRU_API_SECRET=your_api_secret_key
COBRU_REFRESH_TOKEN=your_refresh_token
NEXT_PUBLIC_APP_URL=https://yourapp.com
```
