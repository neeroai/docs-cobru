## Endpoint

`POST /pin/`

## Request fields seen in legacy documentation

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `id_pin` | `number` | yes | Catalog identifier of the PIN product. |
| `operator` | `string` | yes | Provider or operator identifier. |

## Response fields seen in older docs

| Field | Type | Notes |
|-------|------|-------|
| `status` | `string` | Purchase result. |
| `pin` | `string` | Delivered PIN or redemption code. |
| `pk` | `number` | Cobru purchase identifier. |
