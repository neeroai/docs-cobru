## Endpoint

`GET /balance/`

## Typical use cases

- pre-flight checks before transfers or withdrawals
- dashboard balance synchronization
- reconciliation and treasury views

## Response shape seen in legacy docs

```json
{
  "balance": "14179.25",
  "balance_cop": "14179.25",
  "balance_usd": "9995.00",
  "balance_ars": "10000.00",
  "balance_mxn": "96536.31",
  "balance_brl": "89772737.20"
}
```

## Example request

<Tabs items={['cURL', 'JavaScript / TypeScript', 'Python', 'PHP']}>
  <Tab value="cURL">
    ```bash
    curl -X GET https://dev.cobru.co/balance/ \
      -H "Api-Token: $COBRU_API_TOKEN" \
      -H "Api-Secret-Key: $COBRU_API_SECRET" \
      -H "Authorization: Bearer $COBRU_ACCESS_TOKEN"
    ```
  </Tab>
  <Tab value="JavaScript / TypeScript">

    const response = await fetch('https://dev.cobru.co/balance/', {

## Integration guidance

<Accordions>
  <Accordion title="Do not treat the raw response as ledger truth">
    Use balance reads as an operational snapshot, not as the only source of truth for settlement. Pair them with movements and payment status when building reconciliation.
  </Accordion>

  <Accordion title="Cache carefully">
    For dashboards, short-lived caching is reasonable. For payouts or withdrawals, re-read the balance immediately before executing the operation.
  </Accordion>

  <Accordion title="Multi-currency fields">
    The legacy response includes COP, USD, ARS, MXN, and BRL balances. Confirm which balances are actually meaningful for your account type before showing them in production UI.
  </Accordion>
</Accordions>
