# Retiros a banco

## Autenticación
Todos los endpoints requieren:
- **Bearer Token** para autenticación.
- **API Key** para autorización.

Asegúrate de incluir ambos en los encabezados de la solicitud.

---

## Endpoints

### 1. `GET /bank_list/`
Obtiene la lista de bancos disponibles. Si se proporciona un parámetro `country`, los bancos se filtrarán por el país especificado.

#### Solicitud
- **Encabezados**:
  - `Authorization`: `Bearer <token>`
  - `x-api-key`: `<api-key>`
- **Parámetros de consulta**:
  - `country` (opcional): Filtra los bancos por país. Acepta códigos de país de dos.

#### Códigos de país
| *País*        |*2 Letras* | *Moneda*                |
|------------------|--------------|------------------------|
| Argentina        | AR           | ARS (Peso Argentino)  |
| Brasil           | BR           | BRL (Real Brasileño)  |
| Colombia         | CO           | COP (Peso Colombiano) |
| Ecuador          | EC           | USD (Dólar Estadounidense) |
| México           | MX           | MXN (Peso Mexicano)   |

#### Respuesta
- **200 OK**: Devuelve la lista de bancos disponibles.
  ```json
  [
    {
      "id": "1",
      "name": "Nombre del Banco",
      "country": "COL",
      "currency": "COP"
    }
  ]
  ```

### 2. `GET /get_withdraw/`
Obtiene los detalles de un retiro específico mediante su clave primaria (`pk`).

#### Solicitud
- **Encabezados**:
  - `Authorization`: `Bearer <token>`
  - `x-api-key`: `<api-key>`
- **Parámetros de consulta**:
  - `pk` (requerido): Clave primaria del retiro a obtener.

#### Respuesta
- **200 OK**: Devuelve los detalles del retiro especificado.
  ```json
  {
    "id": "123",
    "amount": 1000,
    "status": "Pendiente",
    "created_at": "2025-01-24T12:00:00Z"
  }
  ```
- **404 Not Found**: Retiro no encontrado.

### 3. `POST /cancel_withdraw/`
Cancela un retiro específico mediante su ID.

#### Solicitud
- **Encabezados**:
  - `Authorization`: `Bearer <token>`
  - `x-api-key`: `<api-key>`
- **Cuerpo**:
  ```json
  {
    "id": "123"
  }
  ```

#### Respuesta
- **200 OK**: Retiro cancelado exitosamente.
  ```json
  {
    "message": "Retiro cancelado exitosamente."
  }
  ```
- **400 Bad Request**: Solicitud inválida o el retiro no puede ser cancelado.
- **404 Not Found**: Retiro no encontrado.
